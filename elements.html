Developer Resources

Technical Documentation

This page contains reference material for teams integrating Veltrix-managed infrastructure into their own systems. Our platform exposes a REST API for provisioning, monitoring, and configuration — all authenticated via short-lived tokens issued by our identity service. The examples below use curl but any HTTP client will work. SDKs for Python, Go, and TypeScript are available on our GitHub organization.

Authentication

All API calls require a Bearer token obtained from the /auth/token endpoint. Tokens are valid for 3600 seconds. Pass your client_id and client_secret in the request body. Never expose credentials in client-side code or commit them to version control — use environment variables or a secrets manager such as AWS Secrets Manager or HashiCorp Vault.

Rate Limits

The API enforces per-account rate limits to ensure fair usage. Standard accounts are limited to 120 requests per minute across all endpoints. Enterprise accounts have higher limits configured at the time of contract. When you hit a rate limit you will receive a 429 Too Many Requests response with a Retry-After header indicating when you can resume requests.

Environments

We maintain three environments: production at api.veltrix.io, staging at api-staging.veltrix.io, and sandbox at api-sandbox.veltrix.io. The sandbox uses identical API contracts but returns simulated data and does not incur charges. Use it freely for integration testing.

Versioning

The current stable version is v2. Pass the version in the URL path: /v2/resources. Version 1 will reach end-of-life on 31 March 2025 and will stop accepting requests at that point. Migration guides from v1 to v2 are available in the changelog. Breaking changes are never introduced within a major version.

Webhooks

Subscribe to real-time events by registering a webhook endpoint in your account dashboard. Events are delivered as signed JSON payloads over HTTPS. Verify the X-Veltrix-Signature header on each request using your webhook secret to protect against spoofed events. Undeliverable webhooks are retried with exponential backoff for up to 24 hours.

Glossary

Compute Unit (CU)

The base billing unit for managed compute resources. One CU corresponds to 1 vCPU and 2 GB of RAM running continuously for one hour. Fractional CUs are rounded up to the nearest 0.1 at the end of each billing period.

Availability Zone (AZ)

An isolated data center within a cloud region. Distributing workloads across multiple AZs provides fault tolerance. Multi-AZ deployment is automatically enabled for Enterprise plans and available as an add-on for Standard plans.

Egress

Data transferred out of the Veltrix platform to the public internet or to external cloud providers. Intra-region traffic between your own services is not charged. Egress pricing varies by destination region and is listed in the full pricing table in your account dashboard.

Platform Overview

Region Availability

Veltrix currently operates in six regions: US West (San Francisco), US East (Virginia), EU West (Frankfurt), EU North (Stockholm), APAC (Singapore), and APAC (Sydney). Data sovereignty constraints for EU clients are handled via region-locked accounts that guarantee all data, including backups and logs, remains within the EU boundary. GDPR data processing agreements are available on request.

Each region runs at least two availability zones. Cross-region replication is available as an enterprise add-on for disaster recovery scenarios requiring geographic redundancy. Failover to a secondary region is a manual operation requiring explicit confirmation — we do not perform automatic cross-region failovers to avoid unexpected cost and compliance implications for our clients.

Notice

Scheduled maintenance windows occur on the second Tuesday of each month between 02:00 and 04:00 UTC. Clients will receive notification at least 72 hours in advance via email and in-dashboard banner. Emergency maintenance required to address active security vulnerabilities may occur outside this window with shorter notice.
